## v1.0.90 - 2026-03-31

## 🎉 CodeNote Helper v1.0.90 发布说明

`v1.0.90` 是一次面向**稳定性 + 授权可用性 + 同步可靠性 + 学习效率**的正式整合发布。  

### 🚀 新增与增强 (New & Enhanced)
- **笔记生成交互升级（LeetCode / TorchCode）**：支持“完整笔记 / 仅答疑”双模式，流式生成过程新增“停止生成”，thinking 与可见答案分区展示，长内容场景下可控性更强。
- **仅答疑能力补齐（AI Tutor 2.0）**：不再是空泛建议，而是更像一对一带练：先讲清你这版题解的优缺点，再逐条回应你的卡点与疑问，并给出可直接上手的完整代码参考，让每次提问都更有“实战收获感”。
- **API 域名动态授权模型升级**：从静态白名单转为按用户配置域名动态申请，新增“前置安全说明 + 浏览器原生授权”双层确认链路，权限范围更可控。
- **笔记页可操作性增强**：顶部关键信息与操作区支持 sticky，长文滚动时“保存/导出/打开题目”等关键操作始终可达。
- **动画与细节体验优化**：猫猫动效改为更平滑的渲染方式，减轻卡顿感；笔记与渲染细节进一步打磨。

### ✨ 修复与体验优化 (Fixes & Improvements)
- **Cloud Sync 功能已移除**：`v1.0.90` 正式版不再提供 Cloud Sync 同步能力，避免多通道状态冲突与历史回滚问题。
- **同步策略收敛**：同步链路明确为“本地存储 + 你主动配置的备份方案”，减少隐式自动同步带来的数据不确定性。
- **首次安装授权稳定性修复**：修复首次授权链路偶发“未授权/无法检查权限”问题，完善 runtime 异常识别与可操作提示。
- **多入口一致性**：弹窗与 options 页统一复用同一套 API 域名授权逻辑，减少行为分叉与排查成本。


### ⚠️ 重要提示：权限与同步使用建议
- **API 域名授权**：首次保存新域名时会出现前置说明与浏览器权限弹窗；拒绝授权将不会保存该配置。
- **Cloud Sync 已下线**：若你曾在历史本地版本中使用过 Cloud Sync，请以当前版本实际设置项为准，不要再按旧流程操作。

### 📦 升级建议
- 升级后建议先在“设置与同步”检查当前可用的同步与备份配置是否符合你的预期。
- 若使用页面内 API 生成，建议确认 `API Base URL / API Key / Model` 已保存且对应域名已授权。
- 若你通过 GitHub Releases 离线安装，建议下载并替换为本版本完整包，避免历史脚本残留带来的行为差异。
