## v1.1.3 - 2026-05-04

## 🎉 CodeNote Helper v1.1.3 发布说明

`v1.1.3` 聚焦于 **复习提示增强 + FSRS 参数改进 + 设置同步闭环**。这次继续坚持最小必要改动，不改现有复习主链路，不改 `references`，重点把题目详情页与 popup 复习评分后的提示日期化、FSRS 参数配置入口、以及本地 / JSON / WebDAV 一致的配置存储链路补齐。

### ✨ 新增与增强 (New & Enhanced)
- **评分成功提示补齐下次复习日期**：LeetCode 题目详情页首次加入复习计划、已在计划中的再次评分，以及 popup 🧠 快捷复习，成功后都会展示自然中文提示，并明确显示“下次复习时间为 YYYY-MM-DD”。
- **FSRS 参数配置进入 options 页面**：新增“开启 FSRS 参数配置”开关；只有开启后才显示参数策略区。支持 `高强度 / 正常 / 佛系 / 自定义` 四种模式，其中正常与当前默认参数完全一致，自定义默认值也与当前默认参数一致。
- **自定义参数保持轻量**：本轮自定义只开放 `request_retention` 与 `maximum_interval` 两个关键参数。

### 🛠️ 修复与改进 (Fixes & Improvements)
- **关闭配置时严格回退 v1.1.2 默认行为**：如果用户没有开启“FSRS 参数配置”，系统仍继续采用当前默认参数 `request_retention=0.9 / maximum_interval=365 / enable_fuzz=false / enable_short_term=false`。
- **保存配置后立即批量重算已入计划题目**：切换高强度 / 正常 / 佛系 / 自定义并点击保存后，会立即重算当前已在复习计划中的 LeetCode 题目 `nextReviewAt`，确保后续提示、排序与统计全部跟着新参数生效。
- **同步 / 备份 / 导入恢复补齐 reviewFsrs**：FSRS 配置正式进入本地持久化、JSON 导出导入、WebDAV 备份恢复链路，恢复后另一端也能拿到同样的参数配置。

### 📋 升级建议
- 如果你之前已经在用复习功能，升级后无需迁移历史 `review` 或 `fsrsState` 数据，旧记录会继续兼容。
- 如果你想缩短复习间隔，请先到 `options` 页开启“FSRS 参数配置”，再切换到高强度或自定义；未开启前系统不会改变原有默认行为。

## v1.1.2 - 2026-04-26

## 🎉 CodeNote Helper v1.1.2 发布说明

`v1.1.2` 聚焦于 **ChatGPT 时间轴真实 Edge 兼容修复 + popup 题单体验优化**。这次继续坚持最小必要改动，不做大重构，重点把真实 Edge 历史对话里的 tooltip 回填、点击跳转、初始激活与星标恢复问题补完，同时完善 popup 题单多题目删除、悬浮信息与布局显示这几类体验问题。

### ✨ 修复与体验优化 (Fixes & Improvements)
- **ChatGPT 时间轴只保留用户提问节点**：继续使用 `conversation-turn` section 建模回合，但明确排除 assistant section，不再把回答节点一起画进时间轴。
- **真实 Edge 早期节点 tooltip 回填修复**：针对历史对话打开后早期 `conversation-turn` section 仍在、但 user 子节点未立即挂载的情况，改为结合 turn 结构做谨慎 user 回合推断，并在 hover / 后续挂载时回填真实用户提问，避免长期只显示“第 X 轮提问”。
- **真实滚动容器定位修正**：ChatGPT 时间轴点击节点时优先驱动识别到的实际滚动容器（如 `data-scroll-root` / `group/scroll-root`），不再继续放大 `document.scrollingElement` 误判带来的尾部偏置。
- **点击跳转彻底恢复**：时间轴节点定位锚点改为稳定绑定当前用户回合 `section`，不再错误回退到整段超大祖先容器，修复“点击有高亮但页面完全不滚动”的真实 Edge 问题。
- **初始激活与星标恢复**：初始化完成后立即按当前滚动位置同步 active 节点，避免刚进入历史对话时错误停在第一个节点，导致后部星标看起来“全部消失”，必须滚动或点击后才恢复。
- **popup 题单所有题目都可删除**：普通导入题单、非 Hot100、以及多题目列表全部补齐删除入口，不再出现只有第一条题目可删的情况。
- **题单卡片布局向力扣题目列表对齐**：移除题号单独占行，压紧标题、笔记、删除按钮间距；题单页仍然不显示“下次复习时间”，也不增加 `🧠` 复习按钮。
- **题单 tooltip 统一收口**：所有题单题目统一复用 popup 力扣题目列表同一套悬浮信息构建逻辑，缺少 URL / 难度 / 标签 / 通过状态时不再出现 `undefined`、`null` 或 `[object Object]`。

### 🛠️ 底层与稳定性治理 (Under the Hood)
- **题单条目身份补全**：对未命中 `matchedRecord` 的题单条目，按 `site + problemKey/titleSlug` 补齐可复用的记录标识，统一走现有 popup 删除/笔记链路，不另起新系统。
- **真实 Edge DOM 证据驱动修复**：本轮不再只依赖模拟页面，新增基于真实 Edge 导出的 DOM / HTML / tooltip / click jump 证据文件逐轮定位根因，明确区分“空壳 section、错误滚动容器、错误锚点祖先”三类问题。
- **时间轴点击定位链路收紧**：在已识别滚动容器上显式推进 `scrollTop`，并把 ChatGPT 回合锚点约束在当前用户回合 `section`，确保节点点击定位在自动化环境与真实滚动容器场景中都能落到正确位置。

### 📦 升级建议
- 如果你之前在 Edge 浏览器的 ChatGPT 时间轴里遇到过节点显示不全或者点击节点没有跳转，建议直接升级到 `v1.1.2`。
- 如果你常用 浏览器插件管理题单，本版本会让删除、悬浮信息和列表布局都与力扣题目页保持一致，升级后建议重新打开 popup 检查当前题单显示。

`v1.1.1` 聚焦于力扣复习链路的稳定性与控制台日志降噪，目标是把“同天重复操作”和“历史记录缺 URL 数据兼容”这两类高频体验问题一次收口。

- **复习评分与提醒去重**：对同一道题，当天只允许记录一次评分，不再区分“首次加入”与“到期复习”；提交通过后的提醒也限制为每天最多一次。当天已评分或提醒过的题目不再重复闪烁打断心流。
- **历史记录缺 URL 兼容**：针对早期历史数据缺少 URL 字段的问题，改为优先使用 `site + problemKey + canonicalId + titleSlug` 的组合特征进行回退定位，确保以前刷过的老题目也能正常打开、加载与保存笔记。
- **操作反馈机制优化降噪**：将“导入题单已存在”、“不支持该题单来源”、“坚果云 WebDAV 上传超时”等可预期的常规网络与业务失败，从全局错误级统一降为轻量提示级，避免引发扩展崩溃的误报。

### 🛠️ 底层与稳定性治理 (Under the Hood)
- **复习依赖模块加载防回归**：新增统一静态校验，严格约束本地 HTML 页面加载记录模块时，必须前置同步加载 `review-fsrs.js` 与 `review-domain.js`，从根源杜绝底层复习算法模块未挂载导致的运行报错。
- **扩展重载期日志治理**：统一拦截处理了浏览器扩展重载期间常发的 `Extension context invalidated` 异常，存储与消息链路不再疯狂刷屏 error 日志。

### 📦 升级建议
- 本次为体验与稳定性专项升级，原有笔记和复习调度数据完全兼容不受影响。如果此前遇到了老题目无法打开笔记、或操作坚果云和题单时偶发假死报错，建议直接覆盖升级至本版本。

## v1.1.0 - 2026-04-13

## 🎉 CodeNote Helper v1.1.0 发布说明

刷题不复习等于白刷，这个版本把FSRS间隔复习系统正式带进了工作流，让做过的题真正形成长期记忆。

- **笔记页目录书签**：预览模式下自动从 Markdown 标题(h1-h6)生成目录导航，点击即可平滑滚动到对应位置。目录固定跟随视口，滚到文末也能快速回跳上文，小屏设备自动隐藏以保证阅读区可用。

### ✨ 优化与细节调整 (Improvements)
- **Popup 题目卡片信息更丰富**：tooltip 补充了 引入 **FSRS 间隔重复算法** 来动态计算每道题的 Recall Probability，并且保存当前记忆状态，鼠标悬停就能快速了解复习进度。
- **快捷复习入口**：已进入复习计划的题目旁边新增 🧠 按钮，复习完直接重新评分，不需要跳转到题目页。
- **数据层复习状态扩展**：`problemRecord.review` 中新增完整的复习状态字段，同时引入 `review_rated` 行为类型，为后续更精细的复习调度打基础。

### 📦 升级建议
